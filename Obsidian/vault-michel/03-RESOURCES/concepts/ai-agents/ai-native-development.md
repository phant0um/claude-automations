# AI-Native Development

Tratar AI como fundação de como software é construído, não atalho para codar mais rápido. Frontier teams redesenham workflows.

## 3 fatores multiplicativos

```
AI handling low-judgment work (1.5x) × uninterrupted focus on high-judgment work (1.5x) × instant access to domain expertise (1.5x) = ~3.4x
```

Remove um → gains colapsam.

## Resultados reais (Amazon)

- Pathfinder: 6 engenheiros, 76 dias, projeto de 30 devs/12-18 meses. Commits 2/sem → 40/sem (20x)
- Structured sprint: 556 commits vs 96 baseline (6x), 90 semanas → 24 semanas (4x)
- In-situ: mediana 4.5x, alguns 10x+

## 5 práticas

1. Invest in agent context (steering files, monorepo, conventions)
2. Slow down to speed up (2 semanas slow → compounding)
3. Feed agents instead of babysitting (backlog, parallel, async review)
4. Make intent explicit before code (specs, task decomposition)
5. Shift testing left (agents self-correct locally)
