# Scaling Behavior of Single LLM-Driven Multi-Agent Systems

**Fonte:** arXiv 2606.00655v1 — Fudan University (Li, Gu, Cai, Feng)
**Tese:** MAS performance does NOT scale monotonically with agent count. Follows diminishing returns governed by synergy vs. coordination overhead trade-off.

## Tese Central

A common assumption — "more agents = better performance" — is empirically false. For homogeneous MAS (all agents share the same LLM), performance follows an **inverted-U pattern** with agent count. The framework SIMAS isolates this law by stripping all MAS complexity down to sequential inter-agent communication.

## Argumentos Principais

**O que SIMAS é:**
- Sequential Iterative Multi-Agent System — minimalist architecture
- n agents, T discussion rounds (default 3), sequential chain communication
- Each agent gets unique profile: personality + core beliefs + expertise (auto-generated by LLM)
- History grows monotonically within each round; first agent synthesizes final output
- All agents share identical model weights — only configuration differs (homogeneous MAS)

**Os 7 Findings:**

| # | Finding |
|---|---------|
| F1 | Sufficiently-driven MAS follows inverted-U: performance peaks at optimal n*, then declines |
| F2 | Task type dictates optimal agent count and tolerance for scaling |
| F3 | Overly large agent counts exhibit "pseudo-stability" — consistent failure, not robust collaboration |
| F4 | Performance degradation is caused by coordination overhead, NOT long-context failure (proven via token-padding experiment) |
| F5 | Structured debate topologies (AutoGen GroupChat) raise peak and delay decline but do NOT eliminate the inverted-U |
| F6 | For reasoning tasks, the primary failure mode of minimalist MAS is fragmentation of coherent thought |
| F7 | Collective intelligence is not automatic — it must be architecturally engineered |

## Key Insights

**Prerequisite: base model capability matters first**
Small models (7B/8B) never benefit from MAS — monotonic degradation regardless of n. Effective MAS requires a sufficiently capable LLM above a performance threshold. This is a hard prerequisite, not just a "nice to have."

**Token cost scales quadratically with n in SIMAS**
Adding agents is expensive: quadratic growth in context tokens. This quantifies the overhead side of the synergy/overhead trade-off.

**Task type is the critical moderator**
- Logical reasoning (abstract algebra, formal logic): steep decline, -27.45pp from n=1 to n=8. Reason: reasoning chains are fragile — one broken link derails the solution.
- Fact retrieval (philosophy, global facts): complex patterns, partial recovery possible through knowledge diversity.
- Mixed reasoning (college physics, CS): non-monotonic, domain-dependent peaks.

**Pseudo-stability is not collaboration**
At high n, output variance decreases — but this is because the system consistently fails, not because it robustly succeeds. Mistaking low variance for quality is a diagnostic trap.

**Coordination overhead, not context length, is the culprit**
Token-padding experiment: total context length held fixed across agent counts. Inverted-U persists. This rules out long-context failure as the mechanism. The overhead is semantic — conflicting reasoning paths, information redundancy, sycophantic consensus.

**The Bystander Effect in MAS (cited concurrent work)**
Agents abandon independent reasoning to comply sycophantically with swarm consensus, even when their internal reasoning was correct. This is a social-psychological failure mode orthogonal to coordination overhead.

**U-shaped recovery (Philosophy with Llama 3.1-70B)**
Phase 1 (n=2-3): focused, minimal overhead. Phase 2 (n=4-5): social meta-discourse proliferates, reasoning fragments. Phase 3 (n=8): majority reiterates correct answer, noise statistically suppressed. This is *pseudo-recovery* — depends on pre-training bias, not collaborative synthesis.

## Exemplos e Evidências

**MMLU experiments (Llama-3.1-70B, n=1–8):**
- Abstract Algebra: peak accuracy then -27.45pp decline to n=8
- Qwen2.5-72B on Abstract Algebra (fixed context): 60.78% (n=2) → 72.55% (n=4) → 64.71% (n=8)
- Philosophy: 94.12% (n=1) → 60.78% (n=4) → 74.51% (n=8) [U-shaped]

**AutoGen GroupChat (cross-architecture validation):**
- Abstract Algebra peaks at 75.00% (n=4,5) vs SIMAS peak — modestly higher
- Abstract Algebra collapses to 50.00% at n=6 — inverted-U preserved
- Philosophy: monotonic decline 94.12% → 86.67% regardless of architecture

**AIME 2025 (hardest math):**
- SIMAS fails catastrophically — CoT single-agent consistently wins
- Reasoning fragmentation across sequential turns destroys solution coherence

**Reasoning benchmark comparison (from [5]):**
| Architecture | Llama GSM8K | Llama AIME 2024 |
|---|---|---|
| Naive-CoT | 87.33 | 16.7 |
| SIMAS (minimalist) | ~fails | ~fails |
| AutoGen | 85.21 | 20.0 |
| Multi-Agent Debate | 90.82 | 20.0 |

Structured MAS (AutoGen, Debate) can beat CoT; minimalist MAS (SIMAS) cannot reliably.

**Model architecture matters independently:**
- Qwen2.5-72B reaches peak at n=2 vs Llama-3.1-70B at n=4
- Qwen shows higher performance retention (86.27% vs 60.78%), lower variance
- Model choice cannot be separated from MAS design

**Agent profile ablation:**
Removing personality, beliefs, or expertise shifts absolute performance but does NOT change the inverted-U law. The law is intrinsic to interaction dynamics, not agent configuration.

## Implicações para o Vault

1. **Vault SO agentes paralelos:** This paper justifies the vault's pattern of limited parallel agents (typically 8-9 for batch ingest) rather than running 20+. Coordination overhead is real and quadratic.
2. **Nexus routing decision:** Don't add agents for the sake of agents. The vault already implements this via Karpathy Simplicity First — single-agent when task fits, multi-agent only when ceiling is hit.
3. **Reasoning tasks:** For analysis/synthesis tasks (harness-engineering synthesis, weekly reviews), single CoT agent with good context likely outperforms a SIMAS-style multi-agent setup.
4. **Structured vs minimalist MAS:** AutoGen/Debate patterns with synthesis loops can beat CoT; unstructured multi-agent dialogue (pure sequential chat) typically cannot. Vault agent design should ensure structured synthesis, not just agent proliferation.
5. **Model capability prerequisite:** Smaller models (Haiku) should not be used as the base for MAS — they'll show monotonic degradation. Routing heavy MAS tasks to Sonnet/Opus is architecturally correct.
