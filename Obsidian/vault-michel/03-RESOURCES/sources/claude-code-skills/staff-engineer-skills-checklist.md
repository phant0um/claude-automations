## Tese central
Staff engineer = projetar o sistema CERTO pro negócio, explicar tradeoffs, influenciar múltiplos times, reduzir dor operacional de longo prazo.

## Key insights
- Arquitetura: ADRs, plataforma, build vs buy, decomposição de monolito, multi-region, retrocompatibilidade, technology boring.
- Escalabilidade: cache, queue, partitioning, sharding, replicação, idempotência, retry storms, eventual consistency.
- Confiabilidade: SLO/SLI, error budgets, postmortems, canary, rollback, feature flags, chaos testing.
