## Tese central

Claude Projects são a funcionalidade mais subutilizada do claude.ai: cada projeto tem system prompt, arquivos carregados e memória persistente entre conversas — transformando Claude de assistant episódico em especialista de tempo integral. O efeito composto é que os projetos ficam melhores com o tempo conforme os arquivos de contexto são atualizados.

## Argumentos principais

- Cada projeto funciona como "contratar um especialista em tempo integral" em vez de "contratar alguém uma vez"
- O segredo não está no system prompt (fácil de copiar) mas nos arquivos de contexto: writing samples, notas pessoais, stack técnico, biases — o que torna cada projeto único
- Spend 80% of setup time nos arquivos, 20% no system prompt
- O efeito composto: após 90 dias, os projetos sabem mais sobre seu trabalho do que a maioria dos colegas

## Key insights

- 7 projetos por ordem de impacto diário: Morning Brief (47 min economizados por dia), Content Engine (voz própria via samples reais), Research Lab (memória de pesquisa acumulada), Second Brain (conecta ideias cross-notes), Inbox Zero (triagem em 10 min), Code Helper (contexto de stack persistente), Personal Strategist (thinking partner com conhecimento de biases pessoais)
- Content Engine: 30 melhores tweets + 5 artigos upadas — Claude imita ritmo e estrutura real, não "instrução de estilo"
- Second Brain prompt: busca nas notas antes da base geral, cita fonte explicitamente, sinaliza contradições com notas anteriores, nunca fabrica conexões
- Inbox Zero: classificação em 🔴 ACTION / 🟡 FYI / ⚪ IGNORE com drafts prontos em 120 palavras max, som humano
- Personal Strategist: faz 1 pergunta clarificadora → recomendação direta → o que você não quer ouvir → confiança (High/Medium/Low + porquê); nunca começa com "Great question", nunca termina com "Ultimately it's up to you"

## Exemplos e evidências

- Morning Brief: 47 min economizados todo dia às 7am
- Content Engine: de 2 posts/semana para 7 — mesma energia criativa
- Inbox Zero: de 30 min para <10 min de triagem diária
- Code Helper: eliminou 80% do copy-paste-fix cycle que quebra flow state
- Ordem de implementação recomendada: Morning Brief (Dia 1 manhã, 30 min) → Content Engine (Dia 1 tarde, 45 min) → Inbox Zero (Dia 2 manhã) → Strategist (Dia 2 tarde) → Research Lab/Second Brain/Code Helper (Semana 2)

## Implicações para o vault

O padrão de Claude Projects é o análogo em claude.ai do sistema de skills e agents deste vault. Os prompts de Second Brain e Personal Strategist são referências concretas para melhorar o Nexus e os agentes existentes. O princípio de "arquivos de contexto > system prompt" é diretamente aplicável ao design de novos agents.
