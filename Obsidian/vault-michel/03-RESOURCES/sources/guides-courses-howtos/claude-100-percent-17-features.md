## Tese central

A maioria usa Claude para 10% — abrir chat, digitar pergunta, fechar aba. Os outros 90% são features que transformam Claude de chatbot em sistema que roda partes do trabalho: 17 features organizadas em 4 partes.

## Argumentos principais

**Part 1 — Hidden Features (acesso imediato):**
- **Projects** — memória persistente de context; upload de conhecimento permanente; end ao recomeço do zero
- **Memory** — Claude lembra preferências, projetos, constraints entre sessões
- **Artifacts** — outputs isolados (código, HTML, SVG) que atualizam ao vivo; compartilháveis; evoluem na mesma conversa
- **Analysis Tool** — dados reais: upload CSV/Excel → análise estatística, gráficos, correlações sem instalar ferramentas
- **Extended Thinking** — modo de raciocínio lento para problemas complexos

**Part 2 — Claude.ai Features:**
- Voice mode, canvas, integrations nativas

**Part 3 — Claude API / Advanced:**
- MCP servers para conectar ferramentas externas
- Workflows customizados

**Part 4 — Claude Code:**
- Execução de código, automações, acesso a filesystem

## Key insights

- **"Pick three. Set them up today."** — estrutura pragmática, não overwhelm
- Projects + Memory = dupla que elimina re-contextualização constante
- Analysis Tool: substitui Excel/Python para análise ad-hoc, sem setup
- Artifacts: outputs vivos que evoluem, não snippets mortos para copiar-colar
