## Tese central

A maioria usa Claude da forma mais fraca possível — novo chat, uma pergunta, fecha a aba, recomeça do zero. Transformar Claude em sistema de trabalho requer configuração: memória persistente, contexto de projeto, MCP, arquivos, workflows repetíveis.

## Argumentos principais

**Part 1 — Comportamento Default:**
- Escrever personal instructions (quem você é, o que trabalha, audiência, formato preferido, tom)
- Claude sem instrução = assistente genérico; com instrução = seu assistente

**Part 2 — Projects:**
- Cada projeto = context separado, arquivos permanentes, system prompt próprio
- Uploads: guia de estilo, docs de produto, exemplos de trabalho passado, templates
- Separar contextos por área de trabalho fundamental

**Part 3 — Memória:**
- Construção ativa: dizer explicitamente "remember that..."
- Preferências de formato, deadlines recorrentes, definições de termos específicos
- Sem construção ativa = recomeço do zero toda sessão

**Part 4 — Arquivos e Dados:**
- Upload direto de PDFs, CSVs, imagens, código
- Claude analisa e mantém contexto do documento ao longo da sessão

**Part 5 — MCP e Integrações:**
- Conectar Gmail, Drive, Slack, GitHub, databases
- Claude opera diretamente nas ferramentas em vez de só gerar conteúdo para copiar-colar

**Part 6 — Workflows Repetíveis:**
- Criar templates de prompt para tarefas recorrentes
- Salvar os que funcionam como "skills" reutilizáveis

## Key insights

- **"Stop treating it like a blank chat window, start treating it like a system you configure"** — mudança de mentalidade central
- Problema diagnosticado: sem contexto salvo, sem preferências, sem memória de projeto, sem arquivos, sem workflows
- Solução: configuração incremental que compound com o tempo
