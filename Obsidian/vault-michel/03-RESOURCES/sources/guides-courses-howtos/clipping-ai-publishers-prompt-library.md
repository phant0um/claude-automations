# AI Publisher's Prompt Library

Comprehensive guide to self-publishing with Claude using structured prompt architecture.

## PublishingOS Concept

- Traditional publishing requires mastery of 12 disciplines (market research, copywriting, design, etc.)
- AI-augmented approach: master ONE meta-skill — **prompt engineering**
- Prompt library = operating system for publishing workflow

## Prompt Architecture Framework (5-part)

1. **Role Assignment** — Tell Claude who it is (specific expertise context)
2. **Context Injection** — Provide raw materials (keywords, manuscripts, data)
3. **Task Specification** — Define exactly what's wanted (ranked lists, scored analyses)
4. **Constraint Setting** — Define boundaries (word count, tone, format constraints)
5. **Output Format** — Specify deliverable structure (tables, sections, markdown)

## Library Structure

- 36 prompts total
- Sequential system (complete first book modules 1→6, reuse modularly afterward)
- Organized by publishing workflow phases:
  - **Module II**: Market Intelligence (niche discovery, validation)
  - **Module IV**: Production Line (book creation)
  - **Module VI**: Launch Operations (go-to-market)
  - **Module IX**: Scale Architecture (portfolio expansion)

## Key Insight

Niche selection is most consequential decision → 3-5 hour research process → higher quality than manual research. Get niche right = mediocre book in hungry market wins. Get wrong = masterpiece in dead market fails.
