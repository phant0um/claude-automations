# 35 KILLER Claude Prompts That Do The Work For You

**Autor:** [@coreyganim](https://x.com/coreyganim) — newsletter Build With AI  
**Thread:** https://x.com/coreyganim/status/2049837999534318073  
**Contexto:** 6 meses de uso em workflows de negócios: strategy, content, automation, research.

> [!key-insight] Princípio central
> Claude não é substituto de mecanismo de busca — é uma **ferramenta de raciocínio**. Os 35 prompts forçam estrutura de pensamento antes de Claude responder. O prompt não é a inteligência: é o scaffolding que organiza o raciocínio humano.

---

## Estrutura dos 35 Prompts

### Grupo 1 — Strategy & Analysis (10 prompts)

Para decisões que realmente importam. O padrão comum: forçar articulação do que você **sabe vs. adivinha**.

| # | Nome | Padrão-chave |
|---|------|-------------|
| 1 | Business Model Validation | Risk → Why → Mitigation; identifica lacunas ocultas |
| 2 | Competitive Differentiation Matrix | 5×5 matrix (Price/Quality/Speed/Support/Innovation) |
| 3 | Decision Matrix for High-Stakes | Prioridades do usuário rankeiam opções; reversibilidade explícita |
| 4 | Go/No-Go Project Evaluation | NOW / DELAY / KILL + confidence level 0–100% |
| 5 | Revenue Bottleneck Analysis | 4 alavancas: deal size / cycle / close rate / volume |
| 6 | Customer Feedback Pattern Analysis | Pattern → count → business impact → action |
| 7 | Pricing Strategy Stress Test | "nuclear option se backfires"; justificativa para céticos |
| 8 | Team Scaling Analysis | 90-day roadmap: hire / train / restructure / delegate / automate |
| 9 | Product-Market Fit Validator | Sinais positivos vs negativos; ONE next test |
| 10 | Strategic Pivot Scenario Analysis | 4 cenários (work/fail/competitor/reversible) + 1 métrica decisora |

## Por que estes prompts funcionam (meta-padrões extraídos)

Ver [[03-RESOURCES/concepts/llm-ml-foundations/prompt-engineering-patterns]] para análise completa. Os padrões recorrentes nesta coleção:

1. **Estrutura de saída pré-definida** — Todos especificam o formato de output antes de pedir o conteúdo
2. **Confidence levels explícitos** — Múltiplos prompts pedem 0–100% para forçar calibração real
3. **"ONE thing" constraint** — 7 dos 35 usam restrição de "only one" para forçar priorização
4. **Reversibilidade como dimensão** — Prompts de decisão sempre perguntam se é reversível
5. **Cenário de falha obrigatório** — "Nuclear option", "Year 1 failure", "What I'd lose" aparecem sistematicamente
6. **Grupos A/B opostos** — Email e CTAs geram variantes conservadoras + agressivas para teste real
