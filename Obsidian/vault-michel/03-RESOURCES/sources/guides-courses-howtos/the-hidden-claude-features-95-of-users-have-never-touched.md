## Tese central

A maioria dos usuários usa apenas 20% do que Claude pode fazer; os outros 80% estão em configurações, toggles e features que nunca foram ativados — e a diferença entre os 5% e os 95% não é conhecimento de prompts secretos, mas 10 minutos no menu de configurações.

## Argumentos principais

- 8 features subutilizadas identificadas: Custom Writing Styles, Memory controls, Projects, Artifact Publishing, Extended Thinking, Connectors, Search Past Chats, e Mini-Apps via Artifacts.
- O padrão comum entre todas: nenhuma exige código, nenhuma está escondida — são apenas features que ninguém mencionou para ativar.
- A distinção real entre power users e casual users não é conhecimento de prompts — é configuração deliberada da ferramenta para o próprio workflow.

## Key insights

- Custom Styles (Settings → Styles): Claude aprende tom, ritmo e estrutura de amostras de escrita fornecidas; múltiplos perfis para contextos diferentes.
- Memory (Settings → Memory): viewable, editável e deletável por entrada; bad memory hurts more than no memory — manutenção ativa é crítica.
- Projects: sistema prompt + arquivos + memória por projeto persistente — diferença entre "estranho" e "colega de meses".
- Extended Thinking: toggle que faz Claude raciocinar mais antes de responder — usar para decisões com stakes reais, debugging complexo, análise nuançada.
- Connectors (Settings → Connectors): integra Gmail, Drive, Calendar — Claude para de ser ilha e entra no workflow real.
- Artifact Publishing: output interativo publicável como link sem necessidade de conta do receptor.
- Artifacts como mini-apps: habit trackers, calculadoras, CRMs simples — "software pessoal em minutos".

## Exemplos e evidências

- Habit tracker, budget calculator, decision matrix, content calendar — todos construíveis conversacionalmente sem código.
- "Search and reference past chats": "o que decidimos sobre minha estratégia de conteúdo na semana passada?" vira pergunta real respondível.
- Connectors: "summarize the doc I shared this morning" ou "what's on my calendar tomorrow" passam a funcionar.

## Implicações para o vault

- Memory controls são relevantes para Michel usar o Claude.ai com gerenciamento ativo de memórias (paralelo ao MEMORY.md do vault).
- Projects no Claude.ai são o equivalente consumer ao sistema de agentes com system prompts do vault.
- Extended Thinking é relevante para tarefas de análise complexa — saber quando ligar/desligar é skill prática.
- Connectors (Drive/Calendar) são útil para integração com workflow real do usuário.
