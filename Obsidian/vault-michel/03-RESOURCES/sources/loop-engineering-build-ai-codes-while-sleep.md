## Tese central
Loop engineering = construir sistema que encontra trabalho, entrega ao agente, checa resultado, decide próximo move, e roda enquanto você dorme. 6 pieces: State, Automations, Worktrees, Skills, Connectors, Sub-agents. Pergunta-chave: converge em algo true ou é random walk caro?

## Key insights
- State é a única coisa que a próxima run herda — memory morre com a run, tem que viver em disk
- $47K queimados em 11 dias unwatched — brakes são metade do skill
- Mapa 6 pieces aplicável a Codex e Claude Code
