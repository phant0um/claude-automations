# PM Brain OS — Second Brain para Product Managers

**@PawelHuryn**: segundo cérebro em Markdown para PMs. Anti-vector-DB, grep-able, sem cloud.

## Design

Pasta de markdown files em git repo. CLAUDE.md instrui Claude como usá-los. Agente lê antes de responder, escreve depois, faz sweep semanal.

> "No vector database. No embeddings. No cloud. No auto-tagging. Everything is grep-able. You can open the entire brain in any editor."

## Anti-Pattern de Sistemas de Memória

5 failure modes que PM Brain evita:
1. Acumular sem sintetizar → vira lixão
2. Flatten contradições em consenso → perde o sinal (o conflito era o insight)
3. Drift silencioso da estratégia → decisões apontam pra lá, strategy doc pra cá
4. Perde decision context → sabe o quê, não o porquê
5. Overload de contexto → agente carrega muito, gera output raso

## 5 Design Choices

1. **Epistemic boundaries**: cada claim é tagueado — observation/interpretation/hypothesis/assumption/decision
2. **Maintenance model que funciona**: sweep semanal, evidence stale flagged, contradições preservadas (não resolvidas)
3. **Flag, never gate**: sistema surfaça, PM decide
4. **Decision audit trail**: por que X? quais alternativas? sempre preservados
5. **Working memory vs durable layer**: só promove o que é recorrente, relevante para decisão ou estratégia

## Relevância para o Vault

Filosofia idêntica ao vault-michel: markdown, grep, sem embeddings, CLAUDE.md como firmware do agente.
