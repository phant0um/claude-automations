## Key insights
- "[[@NainsiDwiv50980]]"
- Today, the biggest constraint isn't the model.
- Into the system that decides: - What work gets picked up - Which agent handles it - How it gets verified - What happens next - What information survives between runs Instead of prompting the model directly, you design a system that prompts the model for you.
- The Agent Can Observe Reality A surprising number of developers ask agents to solve problems they cannot see.
- The Hidden Cost Nobody Talks About As loops improve, a strange thing happens.
- A loop that can modify code can introduce vulnerabilities.
- The engineer who manually prompts an agent can improve one task.
- The engineer who designs a loop can improve thousands.

## Minha Síntese
**O que muda:** Prompt engineering está peaking; loop engineering é o próximo — projetar o sistema que prompts o modelo (work routing, verification, memory survival) em vez de otimizar prompts individuais; um engineer de loop melhora milhares de tarefas.

**Conexão pessoal:** Esta é a transição que o vault já está fazendo — CLAUDE.md e skills são loop engineering, não prompt engineering; o Hermes Agent é o loop system.

**Próximo passo:** Focar em melhorar o loop (routing, verification, memory) em vez de gastar mais tempo otimizando prompts individuais no vault.
