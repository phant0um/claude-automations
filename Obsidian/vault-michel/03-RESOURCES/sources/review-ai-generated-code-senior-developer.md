## Tese central
Reviewing AI code é skill própria, diferente de writing code. 35% AI backend code é secure AND correct (2026 study). ~50% ship com known security weakness (Veracode, 100 models/80 tasks). 3 things about AI: optimized for believable answer (not correct), fills gaps com popular patterns, sounds equally sure when wrong or right.

## Key insights
- "Running and being safe to ship are not the same thing"
- Model não pode te dizer why it made choices — guessed based on patterns
- "The model sounds exactly as sure when it is wrong as when it is right. There is no tell."
- Step 1: start with problem, not code
