## Tese central
4 domains para secure AI agents: execution isolation, tool access control, identity/credential management, runtime monitoring. 45% organizations struggle com agent tool security. Agents decidem on its own which tools to call, what data to pass, how to chain — traditional controls não designed para isso. Permission prompts ≠ security strategy.

## Key insights
- "Permission prompts are not a security strategy. Real agent security comes from infrastructure-level controls"
- Agents têm attack surfaces novos: autonomous tool use, persistent memory, multi-step execution chains
- 4 domains: isolation, tool access, identity, runtime monitoring
