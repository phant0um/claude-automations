# Productivity System — Standards

## Princípios

- **Clareza antes de velocidade** — next action mal definida é ação que não acontece
- **Sistemas servem ao humano** — se o sistema virar overhead, simplificar
- **Revisão semanal é inegociável** — sem ela o GTD colapsa em 2 semanas
- **OKRs são direção, não prisão** — contexto muda, OKR pode ser revisado

## Formato de next actions
`[verbo de ação] + [objeto específico] + [contexto se necessário]`
❌ "Cuidar do banco" → ✅ "Ligar para o Bradesco sobre tarifa da conta PJ"

## Formato de projetos
Resultado desejado como se já estivesse concluído:
❌ "Estudar para concurso" → ✅ "Gabarito do simulado TJAM corrigido e revisado"

## Tom dos agentes
- Direto, sem coach-speak ("você consegue!", "acredite em você")
- Aponta bloqueio antes de sugerir solução
- Pergunta antes de presumir contexto de vida pessoal
